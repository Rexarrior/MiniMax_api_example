# Writer Guide вЂ” Visual Novel Scene Format

## Overview

Each story consists of scenes stored in `stories/{story_id}/scenes/*.scn`.

**Workflow**:
1. Writer fills in **descriptive fields** (what the asset should depict)
2. Run asset generation tool в†’ generates actual files and populates **path fields**
3. Scene plays with real assets

## Scene File Format (YAML)

```yaml
id: scene_id                    # Unique scene identifier
title: "Scene Title"             # Display title

# Background вЂ” describes what the scene background should show
background_prompt: "dark misty forest clearing at dawn, ancient trees, fog, ethereal atmosphere"
background: "bg_intro_dark_misty_forest_clearing_at_dawn.jpeg"  # Generated by tool

# Music вЂ” describes or names the background music
music_prompt: "ambient mysterious forest atmosphere"
music: "ambient_mysterious_forest_atmosphere.mp3"  # Generated or provided

dialogue:
  - narrator: "Narration text here"
  # OR for character dialogue:
  - character:
      id: hermit                # Character identifier
      text: "Dialogue text"
      mood: curious             # Character mood/state
      # After generation:
      voice: "voice_hermit_hash.mp3"           # Generated voice file
      image_prompt: "wise elderly hermit with long white beard"
      image: "char_meet_hermit_hermit_wise_elderly_hermit_0.png"  # Generated

choice:
  - text: "Choice 1"
    next: next_scene_id
  - text: "Choice 2"
    next: other_scene_id

# Optional video background for cutscenes
background_video_prompt: "..."
background_video: "cutscene_intro.mp4"
```

## Field Types

### Descriptive Fields (User/AI fills these)

| Field | Description |
|-------|-------------|
| `background_prompt` | AI image generation prompt for scene background |
| `music_prompt` | Description of mood/type for background music |
| `character.image_prompt` | AI image generation prompt for character in this mood |
| `character.voice_prompt` | Description of voice characteristics |
| `background_video_prompt` | AI video generation prompt for video cutscenes |

### Path Fields (Generated by tool, DO NOT edit manually)

| Field | Generated From | Example |
|-------|---------------|---------|
| `background` | `background_prompt` | `bg_scene_name_generated.jpeg` |
| `music` | `music_prompt` or provided | `ambient_mysterious_forest_atmosphere.mp3` |
| `character.voice` | `character.voice_prompt` | `voice_hermit_abc123.mp3` |
| `character.image` | `character.image_prompt` | `char_scene_char_mood_0.png` |
| `background_video` | `background_video_prompt` | `cutscene_scene.mp4` |

## Asset Generation Tool

The tool reads descriptive fields and:
1. Generates/provides actual asset files
2. Fills in path fields next to the descriptive fields

**Important**: Path fields must be adjacent to their descriptive counterparts in the YAML structure.

## Character Definition

Characters are defined in `stories/{story_id}/assets/characters.yaml`:

```yaml
hermit:
  name: "The Hermit"
  default_mood: neutral
  description: "A wise elderly hermit who guards the forest"
  moods:
    curious: "Curious and questioning"
    serious: "Stern and authoritative"
    wise: "Deep in thought"
```

## Best Practices

1. **Always fill descriptive fields first** вЂ” these are the source of truth for what the asset should depict
2. **Never manually edit path fields** вЂ” let the generation tool handle these
3. **Keep prompts concise but descriptive** вЂ” AI generators work best with clear descriptions
4. **Use consistent mood naming** вЂ” reference moods defined in `characters.yaml`
5. **Test after generation** вЂ” verify assets load correctly using `/tests/*` endpoints
