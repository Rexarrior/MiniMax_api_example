# Roguelike — 2D Turn-Based Dungeon Crawler

A classic roguelike game built with **C++17** and **Raylib 5.5**. Features procedurally generated dungeons, turn-based combat, 10 enemy types, and pixel art graphics.

## Features

- Procedural dungeon generation (BSP + Cellular Automata)
- Turn-based combat system
- 10 unique enemy types (skeleton, goblin, orc, rat, spider, slime, demon, ghost, werewolf, dark mage)
- Field of View (recursive shadowcasting)
- Pixel art rendering from sprite sheets
- Message log and HUD
- Multiple dungeon biomes (dungeon, cave, forest)
- 5 floors to conquer

## Building

### Prerequisites

- CMake 3.16+
- C++17 compiler (MSVC, GCC, Clang)
- [vcpkg](https://vcpkg.io/) package manager

### Steps

```bash
cd app_examples/roguelike

# Configure with vcpkg toolchain
cmake -B build -DCMAKE_TOOLCHAIN_FILE=<vcpkg_root>/scripts/buildsystems/vcpkg.cmake

# Build
cmake --build build --config Release
```

### Running

```bash
# From the roguelike directory:
build/Release/roguelike.exe    # Windows
build/roguelike                # Linux/macOS
```

## Controls

| Key | Action |
|-----|--------|
| WASD / Arrow keys | Move / Attack |
| Space | Wait one turn |
| E | Interact (open chests, use portals) |
| Q | Use health potion |
| ESC | Pause / Quit |

## Assets

Pixel art assets are generated by the MiniMax image-01 API. See `examples_python/10_roguelike_assets.py` for the generation script. Generated assets live in `out/roguelike_assets/`.

## Architecture

```
src/
├── core/          — Config, types, RNG
├── assets/        — AssetManager, SpriteSheet
├── map/           — Dungeon, BSP generator, Cellular automata, tile renderer
├── entities/      — Champion, Enemy, Item, Portal
├── ai/            — Turn-based AI, behavior trees
├── combat/        — Damage calculation, hit resolution
├── camera/        — Viewport camera following player
├── ui/            — HUD, message log, menus
├── systems/       — Turn manager, FOV, particles
└── input/         — Input handler
```
